> 文件说明
> 核心功能：作为 DaE 2.0 的稳定验收标准，约束 Prompt 迭代方向，避免每次改 Prompt 又回到顶层 Schema 争论。
> 输入：DaE 挖掘过程中的用户回答、阶段性画像、机读 JSON 与人读说明书草稿。
> 输出：流程级验收标准、字段级拒收标准、证据标注规则，以及实测复盘触发条件。
> 版本：v3.5 RC（2026-03-13），与 Prompt v3.5 RC 对齐。修订：单问制降级为优先目标（非失败红线）；AlignmentCheck 禁止「暂无」式礼貌总结；Lessons 无具体事件必须标 Insufficient。

# DaE v2 Acceptance Criteria

## 一、总原则
DaE 2.0 的目标不是写一份“很像”的画像，而是写一份：
- **可信**：有事实锚点
- **有洞察**：指出结构性矛盾
- **可复用**：能提升下游 AI 的建议质量
- **全景**：不是单一话题切片，而是能跨职业、健身、理财、穿搭等场景复用的主画像

补充说明：
- 基础信息收集不是天然缺点。真正的问题是**问了不用**，或**过早要求用户回答抽象自我定义题**
- DaE 的职责是建档和诊断，不是给行动方案

## 二、流程级标准

### 1. `Phase 1: 快速建档`
- 验收：用户在 5-8 分钟内可以高密度回答基础事实和 1 个主观锚点（当前最想解决的战役/问题）
- 验收：问题应以事实题、基线题为主，允许跳过敏感项
- 拒收：一上来就追问“你的人生使命是什么”“你的底层驱动力是什么”这类抽象问题
- 拒收：问了一堆基线信息，后续输出完全没用上

### 2. `Phase 2: 议题深挖`
- 验收：围绕一个关键当前战役深挖 `Challenges / Drives / Goals / DecisionStyle / Weaknesses`
- 验收：主要采用单问制，不退化为编号列表连问
- **验收：单议题深挖 4-6 轮后，执行了至少一次跨域验证追问，并要求用户提供第二领域的具体事件或代价作为锚点**
- **验收（置信度规则）：`High` 只能来自第二领域有具体事件佐证的跨域确认；口头认同不得标 `High`；用户否认只降置信，不自动登记 Tension**
- 拒收：模型不做追问，直接心理分析或直接给建议
- **拒收：洞察层（Drives/Goals/DecisionStyle/Weaknesses/Tensions）的所有结论均来自单一议题，未获得任何跨域验证**
- **注意：Phase 2 单问为优先目标，非失败红线**——若模型附带 1 个顺带澄清可接受；整轮打包发出所有问题才是真正的失误

### 3. `Phase 3: 全景收网`
- 验收：在输出前显式检查盲区，补齐空白或证据不足的字段
- 验收：最终图谱不是单议题切片
- 拒收：以“自然对话”为名放任重要字段长期空白

### 4. `Phase 4: 输出`
- 验收：只输出画像、诊断和复用格式，不给行动方案
- 拒收：在说明书、AlignmentCheck 或尾部引导里偷渡行业建议、训练建议、投资建议等

## 三、字段级标准

### 1. `Background`
- 拒收：只有标签，没有关键经历
- 验收：至少包含基础身份 + 1-3 个真正改变用户人生轨迹、核心信念或决策方式的关键经历/关系/健康变量

### 2. `Capabilities`
- 拒收：擅长学习、逻辑好、执行力强等抽象赞美
- 验收：能落到具体技能、方法或产出

### 3. `Resources`
- 拒收：人脉广、有平台、有影响力
- 验收：说明资源内容、可触达性、最近调用情况或激活条件

### 4. `Constraints`
- 拒收：时间不够、精力有限、顾虑很多
- 验收：必须量化或具象化，包含不可协商项

### 5. `Drives`
- 拒收：追求自由、成功、意义、自我价值等无排他性口号
- 拒收：用户提供的人格标签（MBTI/九型等）或口号式表达（“人前显圣”“追求清静”）未经行为验证直接写入
- 验收：必须完成三刀验证——①价值链追问（至少两层，触碰历史代价或边界）②Trade-off test（有真实历史放弃事件）③两者均有事实锚点
- 触发追问：`为什么这对你重要？`（追两层）→ `为了维持这个，你最近一次具体放弃了什么？`
- 特殊情况：用户主动自我剖析并给出 Drives 相关表述时，该自述仅视为完成了价值链追问第一层，**Trade-off test 不得豁免**

### 6. `Goals`
- 拒收：只有方向，没有时间与画面；“更自由”“更有影响力”等无法观察的模糊确认
- 验收：必须有时间锚点与可感知感官画面（能描述实现后第一个看到或感受到的具体变化）
- 触发追问：`如果你实现了这个，你第一个注意到的具体变化是什么？你会看到什么、感受到什么？`

### 7. `DecisionStyle`
- 拒收：理性、谨慎、感性等单标签
- 验收：说明风险偏好、信息需求、行动模式，以及可逆/不可逆决策的不同处理方式

### 8. `Weaknesses`
- 拒收：拖延症、完美主义、不擅社交
- 验收：必须包含触发场景、行为链条、后果链

### 9. `Tensions`
- 拒收：泛泛说“有矛盾”
- 验收：必须由两个已确认字段组成结构性冲突

### 10. `Challenges`
- 拒收：写成长期人生难题或与当前阶段无关
- 验收：必须是当前战役层卡点，可作为顾问切入口

### 11. `Lessons`
- 拒收：鸡汤式结论；只有结论没有事件的半截故事（“意识到X不靠谱”但不说清是哪次失败）
- 拒收：将用户的感受、长期体验或笼统认知翻译成结论直接写入字段
- 验收：必须同时具备三要素——**具体事件**（时间/项目/人）+ **当时的判断逻辑**（为什么这样做）+ **修正后的规则**（此后改变了什么做法）
- 触发追问：`那次具体是什么情况？当时你怎么判断的？后来你改了什么做法？`
- **若用户明确表示无具体事件，必须标 `status: Insufficient`；禁止以用户感受或模糊经历推导出结论替代字段**

### 12. `AlignmentCheck`
- 拒收：写成礼貌总结
- 验收：必须指出核心错位、信息缺口、未解矛盾
- 拒收：夹带行动方案、禁忌清单或下游顾问该怎么做
- 拒收：以「使用说明」「给下游 AI 的注意事项」「如何与本用户沟通」等任何格式输出针对下游顾问的操作指导——即使不使用「建议」二字，只要内容是在告诉下游怎么对待这个用户，就是越权
- 拒收：用「暂无」「已对冲」「已成功桥接」等套话替代实质分析——若未发现明显错位，应列举潜在隐性张力或尚待验证的信息缺口；以「暂无严重问题」作为整段 AlignmentCheck 的结论，视同空白条目，不符合验收

## 四、证据标注规则
每个关键判断尽量补三项：
- `evidence`：原始事实、数字、事件、原话
- `confidence`：`High` / `Medium` / `Low`
- `status`：`Confirmed` / `Inferred` / `UserWithheld` / `Insufficient`

规则：
1. 没有事实锚点，不能标 `High`
2. 用户拒答，必须标 `UserWithheld`
3. 信息不足但又必须暂时保留判断时，标 `Inferred` 或 `Insufficient`
4. `AlignmentCheck` 中的结论也要受证据充分性约束
5. 用户明确终止深挖时，**不得以推断填补空缺**，必须在 `AlignmentCheck` 中逐一列出所有未完成字段并注明"以下字段未完成验证，下游建议可信度降低"，然后以 `Insufficient` 状态输出
6. 如果输出是“全景主画像”，至少要解释为什么当前议题与其他维度有关联，而不是只围绕一个话题写满全文

## 五、Prompt 实测复盘触发条件
如果出现以下任一项，必须复盘 Prompt，而不是继续美化输出：

### 1. 假填表
- 对话机械
- 用户明显疲劳
- 问了一堆基础信息，但最终输出没有消化这些信息
- 一上来逼用户回答抽象自我定义题

### 2. 单议题切片
- 最终画像只覆盖某一个话题
- 换到别的顾问场景几乎无法复用

### 2b. 洞察层单域锁定（与"单议题切片"的区别）
- 覆盖层字段（Background/Resources/Constraints）已在 Phase 1 收全，表面看"全景"
- 但洞察层（Drives/Goals/DecisionStyle/Weaknesses/Tensions）的结论均从同一个议题推导
- 驱动力、弱点、决策风格没有经过跨域验证，无法判断是"普遍模式"还是"场景特定行为"
- 这类输出表面完整，实际上对不同场景下游顾问的提升效果大打折扣

### 3. 像算命
- 洞察很像，但说不清依据
- 用户觉得“被说中了”，但下游 AI 用起来不稳

### 4. 像散文
- 说明书好看
- 但长文本和 JSON 难以复用
- 下游 AI 读取后建议质量提升不明显

### 5. 越权建议
- DaE 在挖掘过程中开始回答“那我该怎么办”
- 或在最终输出中偷渡策略建议、训练方案、投资结论
- 或以“使用说明”“给下游 AI 的沟通技巧”等格式，在人读版说明书或 AlignmentCheck 中嵌入针对下游顾问的操作指导

### 6. 洞察层标签上锁
- 用户提供人格标签（INTP/ENTJ/九型等）或口号式表达（“人前显圣”“追求自由”），模型直接写入 `Drives` / `Goals` 字段，未执行价值链追问和 trade-off test
- `DecisionStyle` / `Weaknesses` 字段大量采用用户自报标签，无行为事件佐证
- 画像“有洞察感”，但每条结论追问一句“你怎么知道的”时，答案都是用户自述而非事实锚点

## 六、使用方式
1. 先用本文件校验 `DaE_Skill_Prompt_v2.md`
2. 再用真实用户对话实测
3. 每次迭代先对照本文件判断是流程问题、字段问题还是输出问题
